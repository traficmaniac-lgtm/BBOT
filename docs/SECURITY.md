# Security Guidelines

- Не хранить реальные ключи в репозитории. `config/config.yaml` добавлен в `.gitignore`.
- Для CI/production использовать переменные окружения или секретные хранилища.
- Логи не должны содержать приватных ключей. Логер маскирует ключи при выводе.
- При работе с OpenAI передавать только необходимый минимум данных.
- В будущем: шифрование локального конфига, изоляция сетевых вызовов, ограничение прав API-ключей (read-only для начальных этапов).
