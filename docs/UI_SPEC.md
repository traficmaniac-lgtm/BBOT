# UI Specification

## Технология
- Tkinter, ttk Notebook для вкладок.
- Минимальная зависимость от внешних библиотек.

## Общие элементы
- Верхний статусбар: текущее состояние state-machine и активный конфиг.
- Левый лог-виджет (на вкладке Logs) прокручиваемый.
- Все кнопки отрабатывают действие (мок/лог/статус), нет пустых обработчиков.

## Вкладки
### Dashboard
- Карточки: статус подключения к бирже (mock), статус AI (готов/нет), активная пара, последний AI-профиль.
- Кнопка "Refresh" обновляет резюме из текущего стейта.

### Pairs
- Кнопка "Load fee-free pairs" (мок-данные) → обновляет таблицу.
- Поле фильтрации по строке.
- Таблица пар (тикер, тип, комиссия) с выбором строки.
- Отображение выбранной пары в статусе.

### AI
- Поле/окно предпросмотра промта (read-only, собирается из конфига и пары).
- Кнопка "Run AI analysis" вызывает клиент (mock/OpenAI).
- Панель с JSON-ответом (форматированное поле).
- Кнопка "Apply settings" → переносит параметры в вкладку Trading.

### Trading
- Поля: бюджет, левередж, тейк-профит, стоп-лосс, таймфрейм.
- Статус RUNNING/STOPPED.
- Кнопки Start/Stop запускают/останавливают моковый движок и обновляют стейт.

### Risk
- Поля: max drawdown, per-trade risk %, max concurrent trades.
- Кнопка "Validate" записывает в лог результат проверки.

### Logs
- Прокручиваемый текстовый виджет с уровнями логов и timestamp.
- Кнопка "Export" сохраняет лог в файл.

### Settings
- Поля: exchange API key/secret, OpenAI key, выбор биржи (combobox), флаг testnet.
- Кнопки: Save / Load / Test Exchange / Test OpenAI.
- Отображение активного конфиг-файла.

## State machine привязка
- После загрузки валидного конфига → CONFIGURED.
- После загрузки пар → PAIRS_LOADED.
- После успешного AI ответа → AI_READY.
- Start → RUNNING, Stop → STOPPED, ошибки → ERROR.
